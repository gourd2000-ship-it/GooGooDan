# 제품 요구사항 정의서 (PRD): 말하는 구구단 챌린지

## 1. 프로젝트 개요 (Overview)
- **제품명**: 말하는 구구단 챌린지 (Talking Multiplication Table Challenge)
- **목표**: 사용자(초등학교 2~3학년)가 선택한 단의 구구단 10문제를 화면에 띄워두고 음성으로 빠르게 말하면, 소요 시간과 정답 여부를 즉각적으로 확인해주는 대화형 웹 애플리케이션 MVP를 개발합니다.
- **핵심 가치**: 재미있고 빠른 피드백을 통해 아이들의 구구단 암기력과 반응 속도를 향상시킵니다.

## 2. 타겟 사용자 (Target Audience)
- **초등학교 2~3학년 학생**: 복잡한 조작 없이 직관적이고 시각적으로 즐거운 UI 필요.
- **교사 및 학부모**: 학생들의 학습 진행 상황과 정확도를 쉽게 파악할 수 있는 결과 피드백 필요.

## 3. 사용자 흐름 (User Flow)
1. **설정 (홈 화면)**: 연습할 구구단의 '단'과 문제 제시 '모드(순서대로, 섞어서)'를 선택합니다.
2. **연습 진행 (진행 화면)**: '시작' 버튼을 누르면 타이머가 작동하고, 화면에 10개의 문제가 표시됩니다.
3. **음성 입력**: 학생이 문제를 보며 정답을 마이크에 대고 순서대로 빠르게 말합니다.
4. **종료 및 채점 대기 (로딩 화면)**: 모든 문제를 말한 후 '완료' 버튼을 누르면 녹음이 종료되고, 채점 중임을 알리는 로딩 화면이 표시됩니다.
5. **결과 확인 (결과 화면)**: 총 소요 시간, 정답률, 문제별 채점 결과가 화면에 표시됩니다. 다시 하기 버튼을 통해 재도전할 수 있습니다.

## 4. 핵심 기술 스택 (Tech Stack)
- **프론트엔드**: `React 19.2`, `Vite`, `Tailwind CSS 4.3`, `shadcn/ui`, `zustand` (상태 관리)
- **백엔드**: `Node.js 24.15.0`, `Express.js`, `multer` (음성 파일 처리 미들웨어), `@supabase/supabase-js` (사후 기능 확장 대비 초기 세팅)
- **음성 인식 및 판정**: `Gemini API` (@google/generative-ai)

## 5. 시스템 아키텍처 (System Architecture)
- **클라이언트 (React)**: 마이크 접근 권한을 얻고 `MediaRecorder API`를 사용해 사용자의 음성을 녹음합니다. 녹음이 완료되면 음성 파일(Blob)을 백엔드로 전송합니다.
- **서버 (Node.js)**: 프론트엔드로부터 전송받은 음성 파일과 선택한 구구단 메타데이터를 함께 `Gemini API`로 전송합니다.
- **AI 판정 (Gemini API)**: 멀티모달 오디오 분석을 통해 음성에서 말한 숫자를 추출하고, 제시된 구구단 문제의 정답과 비교하여 최종 채점 결과를 JSON 형태로 서버에 반환합니다.

## 6. 핵심 기능 요구사항 (Functional Requirements)
1. **단 선택 및 문제 생성 기능**: 2~9단 중 선택 가능하며, 선택한 단에 대한 10개의 문제를 생성해야 합니다.
2. **타이머 및 녹음 기능**: 시작과 동시에 밀리초 단위까지 측정되는 타이머가 작동하고 녹음이 시작되어야 합니다.
3. **오디오 업로드 기능**: 녹음된 파일을 서버에서 처리 가능한 포맷(WebM 등)으로 안정적으로 업로드해야 합니다.
4. **AI 채점 기능**: 음성 인식 결과를 바탕으로 사용자가 말한 내용이 실제 정답과 일치하는지 판별해야 합니다.
5. **결과 표시 기능**: 직관적인 UI(예: O/X 표시, 빨간색/초록색 하이라이트)로 결과를 보여주어야 합니다.

## 7. 향후 확장성 (Future Scalability)
- **Supabase 연동 (사후 개발 계획)**: 본 MVP의 기본 기능 구현(음성 판정, 타이머 등)이 완료된 후, 사전에 세팅해 둔 Supabase를 본격적으로 활용하여 다음 기능들을 추가로 개발할 예정입니다.
  - **사용자 관리 (Auth)**: 학생, 교사, 학부모 계정 연동 및 로그인 기능
  - **학습 기록 저장 (Database)**: 각 학생의 구구단 연습 소요 시간, 정답률, 날짜별 기록 등을 DB(PostgreSQL)에 보관
  - **통계 및 랭킹 시스템**: 학급별 랭킹이나 개인별 성장 그래프를 보여줄 수 있는 대시보드 기능 추가
